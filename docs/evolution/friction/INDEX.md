# Friction Index

A friction report is filed whenever the **same** repository pain point is
hit a second time. See `docs/evolution/norms/0004-file-friction-report-on-second-recurrence.md`.

| Date       | File                                  | Summary                            | Status |

| —          | —                                     | (no entries yet)                   | —      |

## Status values

- `open` — friction observed, not yet addressed
- `proposed` — a proposal in `proposals/` covers this friction
- `resolved` — a merged PR has removed the friction
- `dismissed` — declined with a written reason in the file

## How to file a friction report

1. Copy `TEMPLATE.md` to `YYYY-MM-DD-kebab-case-summary.md`.
2. Fill in the **What slowed you down**, **How often**, **Suggested fix**
   sections.
3. Append a row above; never delete a row.
