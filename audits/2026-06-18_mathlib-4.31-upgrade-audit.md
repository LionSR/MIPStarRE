# Mathlib 4.31 Upgrade Audit

Date: 2026-06-18.

This audit records the comparison between the repository's current dependency
pin, `leanprover/lean4:v4.30.0-rc2` with Mathlib `v4.30.0-rc2`, and the
target pin `leanprover/lean4:v4.31.0` with Mathlib `v4.31.0`.

The purpose is not only to make the code compile after the toolchain change,
but also to identify new Mathlib material which can replace or shorten
project-local infrastructure.  The comparison was made against the local
Mathlib checkout by inspecting

```bash
git -C .lake/packages/mathlib diff --name-status v4.30.0-rc2..v4.31.0
git -C .lake/packages/mathlib log --oneline v4.30.0-rc2..v4.31.0 -- <area>
```

where `<area>` ranged over the matrix, finite-sum, probability,
multivariate-polynomial, finite-field, convexity, and C*-algebra directories
used by the LDT formalization.

## Executive Summary

The upgrade is expected to be mostly an API and import-move upgrade rather than
a mathematical restatement of the project.  Mathlib 4.31 adds useful
declaration-level support in six areas relevant to this repository:

1. Matrix reindexing, rank, adjoints, and Gram matrices.
2. Finset and powerset-cardinality bookkeeping.
3. Bernoulli and binomial probability laws.
4. Multivariate-polynomial coefficient and variable lemmas.
5. C*-algebra order, continuous functional calculus, and positive linear maps.
6. Import moves out of older `Data.*` and `AddTorsor.*` locations.

The most plausible immediate replacements are small proof-local substitutions,
not wholesale replacement of the project's paper-specific structures.  In
particular, Mathlib 4.31 does not supply the LDT notions of
`Submeasurement`, `Measurement`, `normalizedTrace`, `tauNormSq`, the
paper-specific consistency quantities, or the completed/projective
submeasurement constructors.  Those definitions should remain project-local.

## Matrix And Hilbert-Space Layer

The repository's quantum layer contains several ad hoc matrix lemmas in
`MIPStarRE/Quantum/FiniteMatrix.lean`,
`MIPStarRE/Quantum/FiniteHilbert.lean`, and
`MIPStarRE/Quantum/ProjectorONB.lean`.

Mathlib 4.31 adds or changes the following relevant declarations.

### Reindexing

Commit `5b9fcab5de`, `feat(LinearAlgebra/Matrix): reindex as AddEquiv and
RingEquiv (#39106)`, adds:

- `Matrix.reindexAddEquiv`;
- `Matrix.coe_reindexAddEquiv`;
- `Matrix.toEquiv_reindexAddEquiv`;
- `Matrix.symm_reindexAddEquiv`;
- `Matrix.reindexAddEquiv_refl_refl`;
- `Matrix.reindexAddEquiv_trans_reindexAddEquiv`;
- `Matrix.reindexRingEquiv`;
- `Matrix.coe_reindexRingEquiv`;
- `Matrix.toEquiv_reindexRingEquiv`;
- `Matrix.toAddEquiv_reindexRingEquiv`;
- `Matrix.symm_reindexRingEquiv`;
- `Matrix.reindexRingEquiv_refl`;
- `Matrix.reindexRingEquiv_trans_reindexRingEquiv`;
- corresponding `LinearEquiv` and `AlgEquiv` coercion lemmas.

These declarations can simplify proofs where the project currently unfolds
`Matrix.reindex` by hand.  They do not directly replace the local theorem
`Matrix.trace_reindex`: Mathlib 4.31 still has no square-matrix theorem with
that exact statement.  The local theorem should remain for now, although a
future cleanup could reprove it from the new equivalence API and propose it
upstream.

### Rank

Commit `dd7fc3b379`, `refactor(LinearAlgebra/Matrix/Rank): generalize
Matrix.rank/LinearMap.rank to semirings (#39158)`, adds a broader rank API:

- `Matrix.rank`;
- `Matrix.rank_zero`;
- `Matrix.rank_one`;
- `Matrix.rank_le_card_width`;
- `Matrix.rank_mul_le_left`;
- `Matrix.rank_mul_le_right`;
- `Matrix.rank_mul_le`;
- `Matrix.rank_vecMulVec_le`;
- `Matrix.rank_unit`;
- `Matrix.rank_of_isUnit`;
- `Matrix.rank_reindex`;
- `Matrix.rank_submatrix`;
- `Matrix.rank_eq_finrank_span_cols`;
- `Matrix.cRank_toNat_eq_rank`;
- `Matrix.eRank_toNat_eq_rank`.

This is relevant to `ProjectorONB.lean`, especially around
`IsProj.trace_eq_rank` and the rank of selected sums of rank-one projectors.
The existing proof uses the spectral decomposition of a Hermitian idempotent.
Mathlib still does not provide a projector-specific theorem
`trace = rank` for complex Hermitian idempotents in the exact form needed
there, so no immediate replacement is available.  The new rank API is still
useful for future proof shortening once the project separates linear-rank
facts from spectral facts.

### Adjoints And Coisometries

Commit `a3f67d0fcd`, `chore(Analysis/InnerProductSpace/Adjoint):
A.toLinearMap.adjoint = A.adjoint.toLinearMap (#40058)`, adds:

- `ContinuousLinearMap.adjoint_toLinearMap`.

This supports the proof of
`Matrix.toEuclideanLin_conjTranspose_mul_self` in
`FiniteHilbert.lean`.  The project theorem remains useful because it is stated
directly for `Matrix.toEuclideanLin`, but its proof should become less
fragile after the upgrade.

Mathlib already contains

- `LinearIsometry.adjoint_comp_self`;
- `LinearIsometry.adjoint_comp_self'`;
- `ContinuousLinearMap.isometry_iff_adjoint_comp_self`;
- `LinearMap.isPositive_adjoint_comp_self`.

These are already close to the proof of
`Matrix.exists_mul_conjTranspose_eq_one_of_card_le`.  There is still no
Mathlib theorem producing a rectangular matrix with orthonormal rows from a
cardinality inequality in exactly this matrix form, so this existence theorem
should remain local.

### Gram Matrices And Vector Operations

Commit `95c9f65b4a`, `feat(Analysis/innerProductSpace/GramMatrix): Add
Gram.posSemidef_of_mapL (#38755)`, adds:

- `Gram.posSemidef_of_mapL`.

Commit `eaccec82ea`, `feat(Data/Matrix/Mul): add diagonal and transpose lemmas

These are relevant to the QXP-layer identities and Gram-positivity arguments

`mulVec`, `vecMul`, and Gram positivity, but they do not replace the
paper-specific QXP definitions.

  `map_ncard_setBernoulli_singleton`.

For this upgrade PR, the prudent choice is not to restate the scalar
Bernoulli proof unless the toolchain change breaks it.  The local statement is
paper-facing and already specialized to the error scalar.  Rewriting it during
the toolchain bump would increase the mathematical surface of the PR.

## C*-Algebra, Positive Maps, And Functional Calculus

The project uses matrix order, positive semidefinite matrices, CFC square

Mathlib 4.31 adds relevant C*-algebra API.

adds:

- `PositiveLinearMap.ofClass`;

These can eventually make the real trace pairing and SDP dual maps more

dual feasibility and complementary slackness conditions.

Commit `77760d482b`, `feat: another joint continuity result for cfc (#40324)`,

Commit `9d9fa2436c`, `feat: connections between order, realPart and

## Import Moves And Expected Breakage

Several Mathlib files were moved or deprecated before 4.31:

- `Mathlib.Data.Real.Sqrt` moved into the analysis hierarchy.
- `Mathlib.Data.Real.Archimedean` and `Mathlib.Data.Real.Hom` moved into
  `Mathlib.Algebra.Order.Archimedean.Real.*`.
- some `Data.Nat` lattice files moved under `Order`.
- additive torsor files moved toward `Algebra.Torsor`.
- duplicate or old declarations deprecated before late 2025 were removed.
- `zero_le'` was deprecated in favor of `zero_le`.
- `Symmetric` as an unbundled definition was deprecated in favor of
  `Std.Symm`.
- `Pi.prod` and related bundled maps were renamed toward `Function.prod`.
- `diff` was renamed to `sdiff` in relevant order/filter contexts.

The repository's direct imports do not currently mention most moved module
names, but transitive imports can still expose changed simp normal forms and
declaration names.  The first build after the upgrade should therefore be read
as an API-repair pass, not as evidence of mathematical drift.

## Replacement Decisions For This Upgrade

The following replacements are appropriate during or immediately after the
4.31 upgrade, if they arise naturally while fixing broken proofs:

- Use `Matrix.reindexAddEquiv`, `Matrix.reindexRingEquiv`, and
  `Matrix.rank_reindex` instead of unfolding `Matrix.reindex` in new proofs.
- Use `ContinuousLinearMap.adjoint_toLinearMap` where the proof currently
  rewrites between continuous and bare adjoints.
- Use `Gram.posSemidef_of_mapL` for QXP Gram-positivity obligations when it
  matches the local shape.
- Use `Matrix.dotProduct_transpose_mulVec` and
  `Matrix.dotProduct_vecMul_transpose` for bilinear vector-matrix identities.
- Use `Finset.card_eq_sum_ite`, `Finset.expect_eq_sum_div_card`, and
  `Fintype.expect_eq_sum_div_card` for new average/cardinality conversions.
- Use the new powerset-card lemmas when exact-cardinality subset families are
  being reconstructed.
- Use `Polynomial.toMvPolynomial_inj` and the new `MvPolynomial.vars` lemmas
  when one-variable specialization or variable-support facts are needed.
- Prefer the new Bernoulli/binomial measure API only when repairing or
  deliberately refactoring `Pasting/Bernoulli/Scalar.lean`; otherwise keep the
  existing finite PMF proof stable.

The following local definitions should not be replaced in this PR:

- `MIPStarRE.Quantum.Op`;
- `MIPStarRE.Quantum.normalizedTrace`;
- `MIPStarRE.Quantum.tauNormSq`;
- `MIPStarRE.Quantum.IsProj`;
- `MIPStarRE.Quantum.Submeasurement`;
- `MIPStarRE.Quantum.Measurement`;
- the LDT `SubMeas`, `ProjSubMeas`, `ProjMeas`, and completion constructors;
- paper-specific average, consistency, collision, and pasting quantities;
- source-labelled theorem statements from the blueprint.

These objects encode the paper's mathematical interface.  Mathlib 4.31 gives
better ambient algebra and analysis, but it does not provide the paper-facing
objects themselves.

## Validation Plan

After recording this audit, the upgrade should proceed as follows:

1. Update `lean-toolchain`, `lakefile.toml`, and the docbuild pins to
   `v4.31.0`.
2. Run `lake update` to refresh `lake-manifest.json`.
3. Fetch the Mathlib cache with `lake exe cache get`.
4. Run `lake env lean` on the first file reported by the build or on
   `MIPStarRE.lean` if the cache build succeeds.
5. Repair failures by using Mathlib 4.31 declarations where they match the
   local mathematical statement, and by preserving project-local definitions
   where Mathlib does not yet supply the paper-specific object.
6. Run `rg -n "sorry|axiom" MIPStarRE` after repairs, then `lake build -q
   --log-level=info` once the local failure frontier is stable.

## Final Upgrade Record

The repository pins were updated to Lean and Mathlib `v4.31.0`, including the
docbuild toolchain and `doc-gen4` pin.  The Lake manifests were regenerated
with `lake update`, and the Mathlib cache was fetched with `lake exe cache get`.

The repair pass preserved the paper-facing interfaces and theorem statements.
The Lean changes are compatibility repairs for Mathlib 4.31: explicit
unfolding of local measurement-family definitions, pointwise function
extensionality where previous `simpa` calls no longer found a definitional
equality, and use of the new or renamed Mathlib declarations where that was a
direct replacement.

Final validation on 2026-06-18:

- `lake build -q --log-level=info` succeeds.
- `git diff --check` succeeds.
- The broad proof-debt scan `rg -n "sorry|axiom" MIPStarRE` only reports
  textual audit/prose occurrences in `MIPStarRE/LDT/Test/AxiomAudit.lean` and
  comments/docstrings.
- The syntax-oriented scan reports no proof-hole syntax; its only matches are
  the prose phrase `axiom-clean` in `AxiomAudit.lean`.

  ```bash
  rg -n \
    "^\\s*axiom\\b|^\\s*unsafe\\s+axiom\\b|\\bby\\s+sorry\\b|:=\\s+by\\s+sorry\\b|\\badmit\\b" \
    MIPStarRE --glob '*.lean'
  ```

Warnings remaining after the successful build are pre-existing style or
deprecation warnings, principally:

- unused `change` warnings in `LowDegreePolynomial.lean`,
  `Pasting/Bernoulli/FromHToG/PaperMoveChain/Moves.lean`, and
  `Pasting/BridgeLemmas/LdSandwichLineOnePoint/OutcomeLemmas.lean`;
- flexible-tactic warnings in `Quantum/FiniteMatrix.lean`;
- header style warnings in `MIPStarRE/Quantum.lean` and `MIPStarRE/LDT.lean`;
- deprecations for `PMF.bernoulli` and `PMF.binomial`.

These warnings do not block the 4.31 upgrade, but they identify good follow-up
work once the toolchain bump has landed.

Follow-up cleanup in this PR replaced all remaining
`Matrix.reindexAlgEquiv_mul` and `Matrix.reindexAlgEquiv_apply` uses by the
Mathlib 4.31 API `map_mul` and `Matrix.coe_reindexAlgEquiv`, respectively.
It also replaced `continuous_finset_sum` by `continuous_finsetSum`.  The
Bernoulli and binomial PMF proof in `Pasting/Bernoulli/Scalar.lean` remains a
larger proof refactor target because the new measure-level API has a different
statement shape.
